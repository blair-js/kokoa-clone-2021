1) 변수 변수는 (문자나 숫자 같은)값을 담는 컨테이너로 값을 유지할 필요가 있을 때 사용한다. javascript에서 변수는 var로 시작한다. var은 변수를 선언하겠다는 것을 의미한다. 변수는 코드의 재활용성을 높여준다. var a=1; alert(a); -> 1 var a=2 alert(a+1); -> 3 var first = "coding"; alert(first+"everybody"); -> codingeverybody var a='coding', b='everybody'; alert(a+b); -> codingeverybody 2) 변수의 효용 //변수를 쓰기 전 alert(100+10); alert((100+10)/10); alert(((100+10)/10)-10); alert((((100+10)/10)-10)*10); // 변수 사용 후 a = 100; a = a + 10; alert(a); a = a / 10; alert(a); a = a - 10; alert(a); a = a * 10; alert(a); 3) 연산자 연산자란 값에 대해서 어떤 작업을 컴퓨터에게 지시하기 위한 기호. ex. a=1 a는 변수, 1은 값(상수), =은 대입연산자 대입 연산자:우항의 값을 좌항의 변수에 대입한다. 비교 연산자: 프로그래밍에서 비교란 주어진 값들이 같은지,다른지,큰지,작은지를 구분하는 것을 의미한다. =,>,<..... 비교 연산자의 결과는 true or false 중의 하나다. -> 블린(boolean) 동등 연산자: == 좌항과 우항을 비교해서 서로 값이 같다면 true, 다르다면 false가 된다. alert(1==2) //false alert("one"=="one") //true *일치 연산자: 좌항과 우항이 '정확'하게 같을 때만 ture alert(1==="1") //false 숫자와 문자이므로 다름... (but, 동등 연산자인 == 즉.. alert(1=="1")은 true로 나옴....) => ===는 숫자1과 문자1을 다르게 인식하고, ==는 양쪽의 값을 같다고 판단한다. 이것이 '정확'의 의미이다. ===는 서로 같은 수를 표현하고 있더라고 데이터 형이 같은 경우에만 같다고 판단하기 때문이다. 결론은 == 연산ㄷ자 대신 === 연산자를 쓰는 것을 강력하게 권함!
