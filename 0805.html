#1 요약정리(java_0805) *OS(운영채제)* pcu, memory, hard disk 로 구성. 주로 우리의 컴퓨터에 존재하는 프로그램 및 파일들은 hard disk에 저장되어있고, 어떠한 프로그램을 더블클릭이나 열기로 실행시키면 그 프로그램이 memory(기억장치)로 복사가 되어 실행된다. => 실행되는 것이 프로세스 임. memory에서 실행 후 해당 프로그램이나 파일에서 어떠한 기능을 수행하게 되면 그 수행상태는 cpu에 복사. *Program* 프로그램이란 문제를 해결하기 위한 모든 절차 및 과정. 즉 솔루션(해결책)을 내놓기 위한 전 과정임. *프로그래밍 언어* low level : 저급 프로그래밍 언어. 컴퓨터가 이해하기 쉽게 작성된 언어. 기계어와 어셈블리어 등이 있다. high level : 고급 프로그래밍 언어. 사람이 이해하기 쉽게 작성된 언어. C언어, JAVA, Basic 등이 있다. JAVA : 객체 지향 프로그래밍 언어 JDK : 자바 개발도구 키트 JRE : 자바 개발 환경 JVM : 자바 가상 머신 => JDK>JRE>JVM (A>B = A가 B를 포함한다.) => 자바 파일을 실행만 한다면 JRE만 있어도 되지만, 자바로 어떠한 것을 개발하기 위함이라면 JDK를 설치해야한다! *컴파일러와 인터프리터* 공통점 : 사람이 이해할 수 있는 고급언어로 작성된 소스코드를 -> 기계가 이해할 수 있는 기계어로 번역해주는 것. 차이점 -컴파일러 소스코드를 한번에 번역. 번역시간이 오래 걸림. 번역을 하면 실행파일(목적파일)이 생성되어 메모리를 사용. -인터프리터 줄 단위로 번역. 번역시간은 빠르지만 실행시간이 느림. 실행파일(목적파일)을 생성하지 않아 메모리를 사용하지 않음. (인터프리터가 직접 실행) 자바 소스코드 파일 A가 있다. (A = xxx.java) A 파일을 컴파일러를 통해 컴파일하면 실행파일(목적파일)이 B가 생성된다. (B = xxx.class) 이 때 컴파일된 B파일을 JVM에서 각 OS마다 실행할 수 있도록 코드로 변환해준다. (코드변환을 위해 JIT 컴파일러와 인터프리터가 동작) => 즉 내가 윈도우에서 개발한 자바파일을, 맥을 사용하는 누군가에게 전달하면, 전달받은자(수신자)의 JVM 설치여부에 따라 실행여부가 결정된다. (JVM 미설치된 경우 OS가 다르면 실행불가. 왜냐 JVM의 인터프리터 기능을 사용하지 못하므로.)
