#요약정리(java_0806, 0809) 1.데이터 타입(데이터 값의 구조와 길이) 정수형 -byte : 1 byte -short : 2byte -int : 4byte -long : 8byte => 데이터 타입은 메모리의 크기를 결정하는 요소이다. 데이터 값이 크다고 해서 메모리를 많이 사용하는 것이 아닌, 데이터 타입이 어떤 형이냐에 따라 메모리 할당 공간이 결정되기 때문이다. => 연산시 주의할 것. 큰 데이터타입을 작은 데이터타입으로 넣을 수 없음. 실수형 -float : 4byte => 데이터 값 뒤에 항상 'f' 붙여줄 것. -double : 8byte 문자형 -char : 2byte -String : 12byte => 문자열 논리형 -boolean : 데이터 값이 true or false ===> 위의 정수형, 실수형, 문자형, 논리형 데이터는 오직 한 값만 가지며 특수한 기능이 없음. ===> 스택 영역에 저장됨. 클래스형 -String ===> 문자열(데이터값)을 변수에 직접 저장하는 것이 아니라, 문자열은 String 객체로 생성되고 변수는 String 객체를 참조하는 것. ===> 즉 객체의 번지를 참조하는 참조타입 이라고 볼 수 있음. (**데이터 값의 번지수를 변수에 저장) ex. String js = "지수"; String jisu = "지수"; => 문자열 리터럴이 동일하면 동일한 String 객체를 공유한다. -new : 객체 생성 연산자 ex. String js = new String("지수"); String jisu = new String("지수"); => 문자열은 같지만 참조하는 객체가 다름!!! 2. 상수(변하지 않는 수) -리터럴 상수 : 이름이 없는 상수 (변수와 달리 데이터가 저장된 메모리공간을 가리키는 이름을 갖고있지 않음) -심볼릭 상수 : 이름이 있는 상수 => 데이터타입 앞에 'final' 이라고 선언 해줄것. 변수를 처음 선언(초기화)할 때만 데이터 값을 지정할 수 있으므로 유의! 3. 형변환(변수나 리터럴 타입을 다른 타입으로 변환하는 것. Casting 이라고 함) : 주로 타입이 다른 값들을 계산하는 경우 사용된다. *자동 정수형 변환 - int->byte 로 변환(큰->작은) 값의 손실이 발생됨. - byte->int 로 변환(작은->큰) 남은 공간은 0 으로 채움. 단, 변환하려는 값이 음수인 경우 1로 채운다. 실수형 변환 - 위 정수형과 동일 *강제 : 변환하고자 하는 데이터형 데이터값 앞에 기재해줄 것. ex. int js = 1; byte jisu = (byte)js; 4. 버퍼(Buffer) : 읽고 쓰기가 가능한 메모리의 배열. direct Buffer : OS가 관리하는 메모리 공간을 이용하는 버퍼 non-direct Buffer : JVM이 관리하는 힙 메모리 공간을 이용하는 버퍼. 5. 입출력 스트림 스트림 : 데이터 이동 통로(실제의 입력이나 출력이 표현된 데이터의 흐름. 즉, 운영체제에 의해 생성되는 가상의 연결고리이며 중간매개자 역할) 데이터 출력 : System.out 객체가 관리 - print(전달 받은 문자열 출력) - println(전달 받은 문자열 출력 후 커서의 위치를 다음줄로 자동으로 이동) - printf(전달 받은 데이터의 서식을 지정하여 출력) 출력 위치제어 : \n(new line:커서의 위치를 다음줄로 이동), \r(carriage return:커서의 위치를 현재 줄의 처음으로 이동), \t(tab:커서의 위치를 다음탭의 위치로 이동) 츌력 서식제어 : %d(정수), %f(실수), %c(문자), %s(문자열) => 출력 서식제어 실수에서 %.1f 로 기재시 소수점 한자리만 출력됨. 데이터 입력 : System.in 객체로 관리 - read() : 표준 입력 버퍼로 부터 아스키 코드 값으로 가져온다. - Scanner : 객체를 사용하여 사용자가 원하는 데이터 타입으로 입력 버퍼로부터 데이터를 가져온다. next, nextLine, nextFloat
